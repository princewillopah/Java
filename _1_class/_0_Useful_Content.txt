    Java File Rules
// ===========================================================================================================================================================================

  => In Java, if a class or interface is declared as public, the name of the file must match the name of that public class/interface. 
        And you can have only one public class/interface per file" applies to both classes and interfaces in Java.
        That is, If a file contains multiple classes or interfaces, only one of them can be declared as public, and the name of the file must match the name of that public class/interface.
  => For classes/interfaces that are not declared as public, there is no restriction on the number of them that can be in a single file, and their names do not have to match the file name. 
        However, it's a good practice to have each class/interface in its own file for better organization and clarity.

    1. Public Class/Interface: The file name must match the name of the public class/interface, and only one public class/interface is allowed per file.

    2. Non-Public Classes/Interfaces: There can be multiple non-public classes/interfaces in a single file, and their names do not have to match the file name. 
       However, it's recommended to have each class/interface in its own file for better organization.
